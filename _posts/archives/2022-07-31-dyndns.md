Let's make our private home server accessible from internet.

## First thing first

I own my domain name `ncenerar.fr` at OVH
and I want `home.ncenerar.fr` and `*.home.ncenerar.fr`
to be point to my home ip address.
Since my IP can change, I need to set up dynamic DNS.
The process varies from providers.
For OVH, go to your dashboard and set up a DynHost with its associated credentials.
Once this is done, I use `ddclient`.

### Installation

```console
apt install ddclient
```

### Configuration

`nano /etc/ddclient.conf`

I use the web with the url of my box because its default web page
provide the information with the currently obtained IP.
You can rely on internet services if you need to.

```ini
# Configuration file for ddclient generated by debconf
#
# /etc/ddclient.conf

protocol=dyndns2
use=web, web=192.168.1.1
server=www.ovh.com
login=the-user-that-I-set
password='MyStrongPassword'
home.ncenerar.fr
```

`nano /etc/default/ddclient`

By default, the `run_daemon` is set to `false`.
Change it to `true` to run in daemon mode.

```ini
# Configuration for ddclient scripts 
# generated from debconf on samedi 27 juin 2020, 18:33:30 (UTC+0200)
#
# /etc/default/ddclient

# Set to "true" if ddclient should be run every time DHCP client ('dhclient'
# from package isc-dhcp-client) updates the systems IP address.
run_dhclient="false"

# Set to "true" if ddclient should be run every time a new ppp connection is 
# established. This might be useful, if you are using dial-on-demand.
run_ipup="false"

# Set to "true" if ddclient should run in daemon mode
# If this is changed to true, run_ipup and run_dhclient must be set to false.
run_daemon="true"

# Set the time interval between the updates of the dynamic DNS name in seconds.
# This option only takes effect if the ddclient runs in daemon mode.
daemon_interval="300"
```

### Verification

```console
ddclient -daemon=0 -debug -verbose -noquiet
service ddclient restart
service ddclient status
```
